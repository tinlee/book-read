```bash
npm install next react react-dom

```

---

## 创建一个 next.config.js

> 在项目根目录创建一个 next.config.js 文件

> 我们引入`@next/mdx`插件来解析 mdx 文件

> 使用`withMDX`方法来配置 mdx 插件

> 通过`pageExtensions`来指定我们的 mdx 文件后缀

```js next.config.js
const withMDX = require("@next/mdx")({
  extension: /\.mdx?$/,
  options: {
    remarkPlugins: [],
  },
});

module.exports = withMDX({
  pageExtensions: ["ts", "tsx", "js", "jsx", "md", "mdx"],
});
```

---

## 设置 codehike

> 设置`@code-hike/mdx`插件，然后添加`remarkPlugins`配置到`next.config.js`文件中

> 这里哦们也可以设置主题颜色，这里使用了官方的 shiki 主题，当然你也可以使用自定义的主题

> 更多主题和插件 https://codehike.org/docs/configuration

```js next.config.js focus=1[10:15],3,8
const theme = require("shiki/themes/nord.json");
const { remarkCodeHike } = require("@code-hike/mdx");

const withMDX = require("@next/mdx")({
  extension: /\.mdx?$/,
  options: {
    remarkPlugins: [[remarkCodeHike, { theme }]],
  },
});

module.exports = withMDX({
  pageExtensions: ["ts", "tsx", "js", "jsx", "md", "mdx"],
});
```

## 创建\_app.js

> 创建一个 pages/\_app.js 文件

> 在 nextjs 的环境中，使用`_app.js`来自定义应用的全局样式

```js app.js
function MyApp({ Component, pageProps }) {
  return <Component {...pageProps} />;
}

export default MyApp;
```
